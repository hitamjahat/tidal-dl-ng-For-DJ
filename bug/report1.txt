Not sure if this helps. I've been hitting the Tidal API with Postman as well and it occurred to me to change auth token to the one I got from the web login. With this token I was able to get the lossless stream.
I overrode the token here
        if self.session.token_type and self.session.access_token is not None:
            headers["authorization"] = (
                self.session.token_type + " " + self.session.access_token
            )
 and successfully got the lossless stream as well.
Looking closer into the JWT, the only difference I see between the web created one and the python-tidal one, is that the web created one has an additional cuk claim. I guess it's a client customer key.
